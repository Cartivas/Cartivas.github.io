Proyecto 01 Kubernetes sobre Proxmox para crear una nube privada orientada a DAW Abrir / cerrar
Proxmox Kubernetes Cilium Ngrok Helm Ubuntu Server

Objetivo

Construir una plataforma donde usuarios con perfil de desarrollo web pudieran trabajar, desplegar y consultar servicios dentro de una infraestructura real, más cercana a un entorno operativo que a un laboratorio aislado.

Arquitectura

El proyecto se apoyó en Proxmox como base de virtualización, nodos Ubuntu Server, un control plane y varios workers unidos con kubeadm, usando containerd como runtime y Cilium como CNI para conectividad y control del tráfico.

Lo que permitía hacer

Desplegar aplicaciones web, publicar servicios, revisar logs, reiniciar despliegues, escalar réplicas y ofrecer un entorno compartido donde otros usuarios pudieran subir y mantener sus servicios con supervisión.

Retos reales

Hubo limitaciones de red para abrir puertos, acceso físico restringido al servidor y ajustes sobre bridges y VLANs. Eso obligó a buscar soluciones más operativas, como el uso de Ngrok para exponer servicios de forma controlada.


Calidad y seguridad

  • Pruebas de escalado y distribución de carga entre nodos.
  • Uso de logs para diagnóstico y seguimiento del clúster.
  • Backups automatizados con cron y rsync.
  • Hardening del acceso con Fail2ban, UFW e IPTables.
  • Despliegues reproducibles mediante YAML y Helm.

Qué demuestra

Este proyecto refleja capacidad para diseñar una solución completa, trabajar con limitaciones reales, documentar el proceso y convertir la infraestructura en un servicio útil para otras personas, no solo en una práctica técnica.

Proyecto 02 Configuración de servidor de correo (MTA) en Windows Server Abrir / cerrar
Windows Server IIS SMTP DNS hMailServer Thunderbird

Objetivo

Implementar un servidor de correo funcional dentro de un entorno Windows Server, capaz de enviar y recibir correos electrónicos en red local, configurando correctamente servicios, protocolos y resolución DNS.

Arquitectura

El sistema se construyó sobre Windows Server utilizando IIS y el servicio SMTP, junto con hMailServer como motor de correo. Se configuraron dominios internos, cuentas de usuario e integración con servicios de red para simular un entorno real.

Configuración técnica


  • Instalación de roles IIS y servidor SMTP.
  • Configuración de DNS para resolución de dominios.
  • Creación de dominios y cuentas en hMailServer.
  • Configuración de SMTP para envío de correos.
  • Configuración de POP3/IMAP para recepción.

Seguridad y red


  • Apertura controlada de puertos en firewall (25, 143, 110, 587, 993, 995).
  • Configuración de autenticación de usuarios.
  • Uso de protocolos seguros (STARTTLS / SSL/TLS).
  • Validación de DNS para evitar problemas de entrega.

Pruebas realizadas

Se verificó el funcionamiento mediante conexiones SMTP y el envío/recepción de correos usando Thunderbird, comprobando la comunicación entre cuentas internas y validando la correcta configuración del servidor.

Qué demuestra

Este proyecto refleja conocimientos en servicios de red, administración de sistemas Windows, configuración de protocolos de correo y resolución de problemas en entornos reales, incluyendo red, seguridad y conectividad.
